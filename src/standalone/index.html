@superfleb/drag-drop
dragDrop and windowMouse demo

The dragDrop() window is initialized with the positioning coordinates of the window, and the dragDrop component will keep track of drag coordinates relative that, so there’s no messy calculation between offset and client positions or keeping track of offsets yourself. You can click anywhere on the window to move it, and it moves relative to where you started.

The underlying function, windowMouse() is also available to use. This does not provide the tracking and position information that dragDrop does, but it does handle all events after the initial mousedown, and it tracks mouse movement on the window (hence the name), not the DOM element, which prevents sticky drags from the mouseup happening off-element and other related issues.

The lack of tracking is why the box with windowMouse snaps itself to align the top left with the mouse cursor. The only position information in the event is related to the mouse pointer. Of course, if this is all you need, windowMouse is available. If you want simple relative drags, there's dragDrop.

The @superfleb/drag-drop package provides interfaces for React (with useDragDrop and useWindowMouse hooks), for Vue (with similar composables), for non-library JavaScript by exporting functions, and this standalone version, which allows you to add fleb-drag-drop.js with a simple <script> tag and use the $dd object in the global scope.

This does use modern ES2019 JavaScript (sorry, Internet Explorer), so a relatively modern browser is necessary.
